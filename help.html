block help information


Question: Do I have to press save? (on the settings page)

Once you press save then all the text you wrote in the inputs and all the buttons that you pressed will be saved to your chrome local storage. It only saves with the save button so I don't have to keep writing stuff to storage constantly. (more efficient and easier)


Question: Why are there different groups? How should I distinguish my groups?

Different groups should have different sites that are at different times. For example, if I wanted to block Twitter only in the morning and YouTube only at night, I would use two different groups. If I wanted to block Twitter and YouTube in the morning and at night, I would use one group. If I wanted to block YouTube in the morning and at night, but Twitter only in the morning, I would use two groups.


Question: What does the "block all sites" button do?

This button accesses all the websites from every group that you have active (selected green for on) and blocks those websites from midnight until midnight.


Question: I made a change on settings, pressed the save button, reloaded, and the page hasn't changed?

Once you click the save button, all your new settings are collected and given to Chrome to be written to the storage. This can sometimes be slow, depending what else Chrome is doing. Whenever you reload the settings page, it gets whatever data is already in Chrome's storage, which can be not up to date. If you edit and save a non-up-to-date settings page, it will overwrite anything you saved previously.


Question: Can I change / add to the photos that show on the blocked site?

YES! First, move your image into the folder titled "images" within the "blockSites" extension folder. Then, put the name of those images and some alt text in the file "blocked.js". There are more instructions of where exactly to write those within that file.


Question: What does the "I changed timezones / recheck" button do?

The way that the code works to see what websites should be blocked is by using the current time and calculating when it will need to next check (when another site should start being blocked or the current ones should become unblocked). This calculation for re-checking is in minutes. Therefore, if you change time zones before the next check should happen, it will be off. The button manually makes it recalculate what sites should be currently blocked.


Question: Why do you have all those permissions? Are you reading and changing information on all sites?

Chrome has designated URL to be a sensitive properity of a tab. Therefore, this extension uses a strong permission, "tabs". Using "tabs" creates a warning that says this extension can "read your browsing history". This permission cannot actually read any past browsing you have done, it can only read the URL information of current sites.

The other permissions are "storage" and "alarms" whose usages are explained in a later question.


Question: I put the same website in sites to block and sites to exclude from blocking, what will happen?

The website will not block. If, in the same time period, you have "youtube.com" in the block list but also you have "youtube.com" in the exclude list, it will not block. The sites being in the same time period is the only thing that matters, not what groups they originated from.


Question: More specifically, what is being stored? What alarms will fire?

There are two things being stored: "config" and "currentBlock". "config" is the user's settings: a list of groups, where each group has the name, active status, sites to block, sites to exclude, times, and days. "curentblock" stores a list of currently blocked websites and a list of currently excluded websites. Alarms fire when a new time starts or the current time block ends, then, "config" is re-analyzed to chose the new websites that should be currently blocked.
